# DOCUMENTO DE REFERÊNCIA DO PROJETO: "BASTIDORES DIGITAIS"

Tema Central: Desvendando a Lógica do Mundo Real através da Engenharia Reversa Conceitual

1.⁠ ⁠A MISSÃO
O mundo digital que usamos diariamente não é mágica; é uma construção complexa de regras lógicas, armazenamento de dados e design estratégico. Neste ciclo de projeto, vocês deixarão de ser usuários passivos para assumirem o papel de investigadores técnicos.
O desafio: Realizar a "Engenharia Reversa Conceitual" de uma funcionalidade famosa do mercado. Vocês deverão usar todo o conhecimento adquirido no 1º ano (Lógica, Banco de Dados, Web, CG e Psicologia) para investigar, criar hipóteses e, finalmente, simular e desenhar como funciona o "cérebro" desses sistemas.
Nota: O objetivo não é programar o algoritmo real, mas sim ter a capacidade lógica de criar um modelo funcional que explique seu comportamento.

2.⁠ ⁠ESTRUTURA DE TRABALHO

    • Equipes: O trabalho será realizado em duplas. Escolham parceiros com quem vocês tenham boa sintonia de trabalho, pois a investigação exigirá debate e construção conjunta.
    • Autonomia: Cada dupla é responsável pelo seu próprio projeto. Não há interdependência entre as equipes.

3.⁠ ⁠SELEÇÃO DOS OBJETOS DE ESTUDO
Para garantir que cada dupla trabalhe em algo que desperte sua curiosidade genuína, os temas não serão sorteados. Vocês poderão escolher o objeto de estudo a partir da "Lista de Investigação" fornecida pelos professores (Anexo I deste documento).
Regra de Ouro da Seleção: Para garantir a diversidade de assuntos na nossa mostra final, cada tema da lista possui um limite máximo de 02 (duas) duplas. A seleção ocorrerá em sala de aula, e as vagas serão preenchidas conforme a manifestação de interesse das duplas. Sejam rápidos e estratégicos na escolha!

4.⁠ ⁠O PRODUTO FINAL: O "DOSSIÊ DIGITAL"

Cada dupla deverá construir e apresentar um Dossiê de Investigação, que será um site estático (HTML/CSS) que organiza e apresenta as conclusões técnicas do grupo. Este dossiê deve obrigatoriamente conter:
  
  1. O FLUXOGRAMA LÓGICO (O Coração do Projeto): Um diagrama detalhado e complexo que tenta explicar todas as regras de decisão da funcionalidade escolhida (Aplicação de Lógica de Programação).
    2. O MODELO DE DADOS (A Estrutura): O Diagrama Entidade-Relacionamento (DER) conceitual que mostra quais tabelas e campos seriam necessários para guardar as informações que o sistema usa (Aplicação de Modelagem de BD).
    3. O RAIO-X DA INTERFACE (O Visual): Telas do sistema recriadas (Mockups), com anotações técnicas apontando onde cada informação do banco de dados aparece para o usuário (Aplicação de Computação Gráfica).
    4. ANÁLISE DE IMPACTO (O Fator Humano): Um texto breve analisando por que essa funcionalidade funciona tão bem do ponto de vista do usuário. Que gatilhos mentais ela usa? (Aplicação de Psicologia).

5.⁠ ⁠CULMINÂNCIA
O projeto será encerrado com uma Mostra Interativa de Tecnologia, onde cada dupla ou trio terá uma estação para apresentar seus "Dossiês" aos convidados, explicando a lógica oculta dos sistemas que todos usam.
